# vue

# VUE模板语句使用规范

  1. 条件 （v-if , v-else, v-if-else, v-show）合理使用 v-if 及 v-show
  2. 循环 （v-for、 :key）
  3. 样式 （:style、:class）
  4. 事件绑定 （v-on、@）
  5. props传递 （加类型判断，不能直接数组接收）
  6. 其他优化细节（v-text、v-html、v-once）

# 组件模板开发

  1. 尽可能语义化标签，使结构更加信息。
  2. 大段功能模块都需要明确中文注释（要求简短明确）
  3. 在代码结构很长的情况下，需要分割线注释批注
  4. 模板语法中的逻辑尽可能在filters、computed、methods中处理
  5. Tab缩进，四个空格，层次要求分明

# 组件样式开发

  1. 基本上处于scoped 私有域开发，CSS预处理器选择less
    • 原因：强大、方便、流行、混合书写、变量、语言函数等
  2. 样式编写方式
    • 函数式编写样式 ，在common目录下封装好对应常用的方法，直接传参
    • 风格：横向编写，分号分隔，禁止大括号，背景图片统一相对路径调用
  3. 如有其他想法可以更换成Sass或stylus预处理器也可以

# 组件脚本开发

  1. scoped私有域下开发
  2. 和模板结构处于同一文件开发
  3. 生命周期及其默认结构编写顺序和规则
    • data -> props -> 生命周期函数 -> computeds -> watch -> methods -> …
    • props 统一使用对象模式，且设置好props属性默认值和类型

# 注释规范

  1. TEMPLATE结构内容注释
    • 大区块之间需要回车换行，且有有单独的中文注释
  2. STYLUS注释
    • 每个大区块的样式的需要有单独的中文注释
    • 每个大区块样式之间需要回车换行
    • 在STYLUS自定义函数库文件类似于mixin.styl，则需要对每个语言函数进行单独的批注，或者一些功能类似的语言函数可以根据功能分类注释
  3. SCRIPT注释
    • 尽可能多用单行注释，注释需要与被注释的地方对齐
    • 生命周期created()、mounted()下的所有方法调用需要单独注释，methods里面涉及接口调用的方法一定要注释，filters里面的过滤器需要注释说明功能

# 命名规范

  1. 组件文件夹命名：
    • 按照功能英文命名，过长则用 ” - ” 连接
    • 其内部的组件名称和样式名称与文件夹同名
    • 其风格与微信小程序组件文件夹命名保持一致
  2. 静态资源文件夹static命名：
    • 英文命名，过长则用 ” - ” 连接
    • 其主目录需要创建一个解释文件（annotation.txt/annotation.md），在这个解释文件中使用中文批注好每个目录的内容，以及每个目录正在被哪些组件调用
  3. 图片文件命名：
    • 如果是精灵图，则需按功能命名
    • 如果是列表渲染图片，则需要按照1-100编号命名
    • 如果是ICON图片，则需要添加 ”icon-”前缀

# 解释文件

  1. 定义：
    • 一个对当前目录下所有的文件夹的一个解释性文档，中文批注每个文件夹下的组件功能或者资源类型
    • 如果是资源类型文件夹，则还需批注调用该文件夹的组件名称和路径
  2. 名称：
    • 统一命名annotation.txt/annotation.md
